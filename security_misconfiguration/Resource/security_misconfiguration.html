Security Misconfiguration

On parle de Security misconfiguration quand un site utilise des identifiants par défaut (admin/admin par exemple), un mauvais chiffrement des mots de passe ou quand le serveur est mal configuré et donne accès a des éléments sensibles. Cette faille est surtout provoqué l’erreur des développeur et leur manque de connaissance à ce sujet.

Reproduction

  1. Le site comporte un dossier /whatever/ accessible
  2. Lorsque l’on se rend dessus, on peut télécharger le fichier Htpasswd
  3. Le fichier contient root:437394baff5aa33daa618be47b75cb49. La deuxieme partie est un hash md5 étant égal à qwerty123@. Il s’agit donc d’un mot de passe faible.
  4. Une page /admin est cachée sur le site, et il suffit de s’y connecter avec root / qwerty123@ pour obtenir le flag

Risques

  • Évidemment, une faille de configuration comme celle ci peut mener a des fuite de données importantes
  • Possibilité pour un attaquant de gagner en privilèges (comme c’est le cas ici)
  • Exposition de données sensibles

Fix

  • Souvent mettre a jour les technos utilisées
  • Adopter les bonnes pratiques de développement
  • Former en interne sur les conséquences de ces erreurs
