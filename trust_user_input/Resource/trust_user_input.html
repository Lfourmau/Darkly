Trust user input

Dans presque toutes les applications aujourd’hui, l’utilisateur est amené a interagir avec le site, et a envoyer des données au back. Si le développeur fait confiance aux inputs de l’utilisateur, ce dernier peut envoyer des données erronées au back sans problème.

Reproduction

  1. On se rend sur la page Survey
  2. Inspecter l’élément sur l’un des inputs de vote
  3. On remarque que l’on peut modifier la value du vote. On passe donc d’un chiffre entre 1 et 10 a 9999
  4. On vote pour obtenir le flag

Risques

  • L’utilisateur peut effectuer des actions interdites (donner une note de 999, par exemple)
  • Certains utilisateurs pourraient avoir accès a des informations qui ne leur sont pas destinées. Par exemple si l’utilisateur demande les infos d’un autre utilisateur et qu’aucune vérification n’est faite pour vérifier son identité, il a accès a tous les utilisateurs.
  • Dans le pire des cas, le serveur pourrait crasher si l’utilisateur envoie une donnée qui n’est pas du tout celle attendue

Fix

  • Ajouter des vérifications coté Back-end pour ne pas faire confiance a l’utilisateur et confirmer que l’input correspond aux règles définies
